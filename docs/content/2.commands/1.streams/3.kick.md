## Commands Overview

All Kick stream commands start with `/streams kick`

| Command | Description |
|---------|-------------|
| [`add`](#adding-a-kick-streamer) | Add a Kick streamer to receive notifications for going live |
| [`remove`](#removing-a-kick-streamer) | Remove a Kick streamer alert subscription |
| [`edit`](#editing-kick-settings) | Edit config for a Kick streamer alert |
| [`test`](#testing-notifications) | Preview notification appearance |
| [`details`](#viewing-streamer-details) | Show the current configuration for a Kick streamer |

## Adding a Kick Streamer

Setup notifications to receive alerts when a Kick streamer goes live:

```
/streams kick add <streamer> <discord-channel> [ping-role] [live-message] [offline-message] [cleanup]
```

### Parameters

::field-group
::field{name="streamer" type="string" required}
The Kick username of the streamer 
::
::field{name="discord-channel" type="text-channel" required}
The Discord channel where notifications will be posted
::
::field{name="ping-role" type="role"}
Role to mention when the streamer goes live
::
::field{name="live-message" type="string"}
Custom message for when streamer goes live - Refer to the [Customizing Messages](#customizing-messages) section for available variables
::
::field{name="offline-message" type="string"}
Custom message for when streamer goes offline - Refer to the [Customizing Messages](#customizing-messages) section for available variables
::
::field{name="cleanup" type="boolean"}
Default to `false` - Auto-delete the live notification when the stream goes offline.
::

## Testing Notifications

Preview what your notifications will look like:

```
/streams kick test <streamer> [message-type] [multistream] [global]
```

### Parameters

::field-group
::field{name="streamer" type="string" required}
The Kick username of the notification you want to test
::
::field{name="message-type" type="string"}
Default to `live` - Test `live` or `offline` notification 
::
::field{name="multistream" type="boolean"}
Default to `false` - Test as multistream notification if it is setup
::
::field{name="global" type="boolean"}
Default to `false` - Display the message to everyone as if the streamer just went live (will ping people so use with caution)
::

## Customizing Messages

### Available Variables

Use these variables in your custom messages:

- `{{name}}` - The name of the streamer
- `{{url}}` - The URL for the stream
- `{{everyone}}` - @everyone mention
- `{{here}}` - @here mention
- `{{game}}`/`{{category}}` - The game or category of the stream (live only)
- `{{timestamp}}` - The time the stream started/ended
